# Web feature domains

SaaS UI modules will live here. **No routes or components are registered yet.**

| Folder | Future feature |
| --- | --- |
| `projects/` | Saved sites, default URLs, project settings |
| `organizations/` | Tenant switching, org profile |
| `teams/` | Members & roles |
| `history/` | Past scan runs & report archive |
| `schedules/` | Recurring scans |
| `billing/` | Plan, usage, invoices |
| `notifications/` | Delivery preferences |

Public scanner UX stays in `views/`, `components/`, and `services/scan/`.

See [ADR 006](../../../../docs/architecture/decisions/006-feature-folders-in-web.md).
